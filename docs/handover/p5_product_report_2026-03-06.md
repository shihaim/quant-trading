# P5 제품 보고서 2026-03-06 (기획 + 운영)

- 작성일: 2026-03-06
- 대상: 기획, 프로덕트, 운영
- 범위: 인증 UX 마무리 및 Bot Control API 계약 정렬

## 1) 목표

일일 운영을 막던 남은 UX/계약 공백을 해소합니다.

- 수동 액세스 토큰 입력 흐름 제거
- Bot 제어를 인증된 사용자 스코프 계약으로 이동
- 전용 Control 페이지의 레거시 Bot 엔드포인트 FE 브리지 fallback 제거

## 2) 이번에 완료한 내용

### 2.1 인증 흐름 앱 내 통합 완료

- 로그인/회원가입 페이지가 `apps/web/app/login`, `apps/web/app/signup`에 활성화되었습니다.
- 인증 성공 시 `access_token`이 자동 저장되고, 목표 페이지로 자동 이동합니다.
- 보호 페이지는 라우트 가드로 비인증 사용자를 로그인 페이지로 리다이렉트합니다.

운영 관점 효과:

- 운영자가 토큰을 복사/붙여넣기 할 필요가 없어짐
- 첫 로그인 온보딩 마찰 감소
- `/api/me/*` 기반 기본 접근 흐름이 더 안전하고 일관됨

### 2.2 Bot Control 계약을 사용자 스코프 인증 계약으로 정렬

백엔드 계약:

- `GET /api/me/bot/status`
- `POST /api/me/bot/start`
- `POST /api/me/bot/stop`

프론트 Bot Control 페이지는 위 계약을 직접 사용합니다.

운영 관점 효과:

- 제어 액션이 인증 사용자 스코프와 정합됨
- FE6이 더 이상 레거시 404 브리지 동작에 의존하지 않음
- 기획 QA 및 운영 런북 기준에서 동작 예측 가능성이 향상됨

## 3) 수용 기준 점검

완료 및 검증:

1. 로그인/회원가입 후 수동 토큰 입력 없이 인증 흐름이 동작함
2. Bot status/start/stop 호출이 `/api/me/bot/*`로 정상 동작함
3. 전용 Bot Control 경로에서 FE 브리지 fallback이 제거됨
4. 회귀 테스트 통과 (`81 passed`)

## 4) 현재 제품 상태

- `2026-03-03` 프론트 MVP 배치가 사실상 완료되었습니다 (`P1-FE1` ~ `P1-FE6`).
- `2026-03-04-v2-foundation`은 완료 상태이며, Bot 제어 인증 계약이 실제 사용 중입니다.

## 5) 현재 제한 사항

- 레거시 엔드포인트 (`/api/bot/enable`, `/api/bot/disable`)는 2026-03-31 sunset 기준으로 retired 처리되며, 서버에서 `410 legacy_endpoint_retired`를 반환합니다.
- 공식 Bot 제어 계약은 `/api/me/bot/start`, `/api/me/bot/stop`입니다.
- Home의 빠른 토글 경로는 레거시 호출을 사용하지 않도록 별도 하드닝 티켓에서 정리해야 합니다.
